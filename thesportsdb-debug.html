Issue 1: Live games missing from /livescore endpoints

4 out of 6 tested games are completely absent from both /livescore/all and /livescore/{sport}.

These are real NBA/NHL games that kicked off 1-3 hours ago. The /livescore endpoints return other games from the same leagues at the same time (Grizzlies vs Rockets, Thunder vs Bulls) but omit these 4.

Pattern observed: Event 2358430 (Celtics vs Hawks) was briefly present in the feed at ~01:22 UTC showing Q1 9-25, then disappeared while the game was still in progress. This suggests games are being intermittently dropped mid-game.

Event 2348033 (Rangers vs Blackhawks) is confirmed finished (FT 6-1 per /lookup) but never appeared in any livescore feed during its entire duration.
Issue 2: /lookup/event returns stale data during live games

3 games that ARE live (confirmed by /livescore feeds showing Q2/Q3 scores) return strStatus: "NS" and intHomeScore: null from the /lookup/event/{id} endpoint.

This is a separate issue from Issue 1 — even when the livescore feed works correctly, the lookup endpoint doesn't reflect the current game state. We understand lookup may have different update schedules, but NS for a game in Q3 is a significant delay.
Question: UTC / Timezone handling?

All kickoff times are stored in UTC. We notice the missing games all have kickoff times between 23:00-23:30 UTC (which is 00:00-00:30 CET, midnight crossing). Could the livescore feed have an issue with games that cross the UTC midnight boundary? The games that DO appear (Raptors at 00:30 UTC, Nuggets at 01:00 UTC) kicked off after midnight UTC.
