# UpDown — User Guide

UpDown is a parimutuel prediction platform on Solana. Predict crypto prices, football matches, and real-world events — all on-chain with USDC.

---

## Quick Start

1. **Connect Wallet** — Click "Connect Wallet". Choose embedded (no extension) or external (Phantom, Solflare).
2. **Pick a Market** — Browse Crypto, Sports, or Prediction markets.
3. **Make a Prediction** — Select your side (UP/DOWN, Home/Away, Yes/No), enter USDC amount, confirm.
4. **Wait for Resolution** — The pool resolves automatically when the event ends.
5. **Claim Winnings** — Go to Profile and claim your payout.

## Market Types

### Crypto Pools
Predict whether **BTC**, **ETH**, or **SOL** will go UP or DOWN within a timeframe.

| Asset | Timeframes |
|---|---|
| BTC | 3m, 5m, 15m, 1h |
| ETH | 3m, 5m, 15m, 1h |
| SOL | 3m, 5m, 15m, 1h |

- A **strike price** is captured at pool creation
- A **final price** is captured at pool end
- If final > strike → **UP wins**. Otherwise → **DOWN wins**
- Interactive price chart with line/candlestick views and 8 intervals

### Sports Markets
Predict football match outcomes: **Home**, **Away**, or **Draw** (3-way).

**Leagues:** Champions League, Premier League, La Liga, Serie A, Bundesliga, Ligue 1, Brasileirão

- Fixtures synced from football-data.org API every day
- Results checked every 5 minutes during match windows (kickoff to kickoff+3h)
- Pools resolve automatically when the match finishes
- H2H analysis generated by AI for each match
- Team crests and match info displayed on cards

### Prediction Markets (Polymarket)
Predict real-world events: **Yes** or **No** (2-way). Powered by Polymarket data.

**Categories:**
| Category | Examples |
|---|---|
| **Politics** | Elections, policy decisions |
| **Geopolitics** | Military actions, diplomacy |
| **Culture** | Entertainment, pop culture |
| **Finance** | Commodity prices, market caps |

- Data synced from Polymarket Gamma API
- Odds chart shows global market probability over time (CLOB API, 30s auto-refresh)
- UpDown odds chart shows our platform's predictions in real-time via WebSocket
- Toggle between Polymarket and UpDown data sources
- Market Rules and Context tabs show resolution criteria
- Pools resolve when Polymarket marks the market as resolved (checked every 10 min)

## How Pools Work

Every pool follows the same lifecycle regardless of type:

| Status | What Happens |
|---|---|
| **JOINING** | Pool is open for predictions. Place your stake. |
| **ACTIVE** | Predictions locked. Waiting for the event to end. |
| **RESOLVED** | Winner determined. Claims enabled. |
| **CLAIMABLE** | Winners can claim payouts via blockchain transaction. |

All pools use **PDA vaults** on Solana — USDC goes into an on-chain escrow account controlled by the program, not the authority wallet. This is trustless: funds can only move according to program rules.

---

## Odds & Payouts

UpDown uses **parimutuel** odds — payouts depend on how much is staked on each side.

**Formula:**
```
Your Payout = (Your Stake / Total on Winning Side) × Total Pool − Fee
```

**Example:**
- $100 on UP, $50 on DOWN → Total Pool = $150
- UP wins → Your payout: ($100 / $100) × $150 = $150 − 5% fee = **$142.50**
- DOWN wins → Their payout: ($50 / $50) × $150 = $150 − 5% fee = **$142.50**

> If everyone bets the same side (no opponents), all bets are **automatically refunded**.

## Tournaments

Bracket-style elimination tournaments where players compete round by round.

### How It Works
1. **Register** — Pay USDC entry fee to join (goes to tournament vault PDA)
2. **Predict** — Each round, predict outcomes for the matchday fixtures
3. **Advance** — Most correct predictions wins the bracket match
4. **Win** — Last player standing wins the prize pool (minus 5% platform fee)

### Tournament Types
- **Crypto** — Predict closing prices each round
- **Sports (Predict the Matchday)** — Predict Home/Draw/Away for real football fixtures + total goals tiebreaker

### Scoring (Sports)
1. Most correct predictions wins
2. Tiebreaker: closest total goals prediction
3. Still tied: first to predict wins

### Prize Distribution
- Prize pool = sum of all entry fees
- Winner receives 95% (5% platform fee enforced on-chain)
- Tournament vaults use PDA escrow — trustless, funds never touch authority wallet

## Data Sources & Sync

### Fixture Cache (Sports)
- Daily sync at 04:00 UTC — fetches 14 days of fixtures per league (~6 API calls)
- Match window polling every 5 min — only checks fixtures currently in play
- Pre-match refresh every 30 min — re-checks fixtures kicking off within 1 hour
- All consumers (pools, tournaments, admin) read from cache, never from API directly
- Reduces API calls from 500+/day to ~30/day

### Polymarket Sync
- Bulk sync every 6 hours — top 200 events by volume
- Resolution poll every 10 min — checks for UMA resolution
- Cleanup daily at 05:00 UTC — removes expired/resolved markets
- Price history from CLOB API (up to 28 days, hourly granularity)

### Price Data (Crypto)
- Real-time prices via Pacifica WebSocket
- Candlestick data for charts (1m to 1D intervals)
- Strike/final prices captured at pool creation/resolution

## On-Chain Architecture

### Solana Program (Anchor)
All funds are held in PDA vaults controlled by the on-chain program. Instructions:

**Pool lifecycle:** `initialize_pool` → `deposit` → `resolve` → `claim` → `close_pool`

**Tournament lifecycle:** `initialize_tournament` → `register_participant` → `claim_tournament_prize` → `close_tournament`

**Cancel flow:** `cancel_tournament` → `refund_participant` (authority-signed, returns entry fees)

### Security
- PDA vaults — funds never touch authority wallet
- Co-signed claims — user + authority must both sign (prevents fee manipulation)
- Fees enforced on-chain (5% for tournaments, level-based for pools)
- DB-first pool creation — prevents on-chain orphans from crashes
